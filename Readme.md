  ```json
  {
    "type": "FeatureCollection",
    "features": [
      {
        "type": "Feature",
          //地区名字
        "properties": {
          "name":"testPlace"
        },
         //地区块
        "geometry": {
            //面
          "type": "Polygon",
            //包裹的经纬度
          "coordinates": [
            [
              [
                55.8984375,
  ```

#### 使用技术与思路

##### 世界地图图表

- 图表使用echarts进行绘制,世界地图是一张map图,每个国家是一个item
- 其中城市是一个点(point),检测到要选择城市时,这个point和国家(item)同时进行选择
- 绘制地图使用geoJson格式,详解world.json文件
- 图表响应事件思路
  - 选择国家标红
    - 检测到用户选择国家后,重新更新图表的options,对图表进行重绘,其中将选中国家的颜色设置为红色,重新渲染显示
  - 选择国家闪烁
    - 检测到用户选择国家后,重新更新图表的options,同时每隔1s将选择国家的边框颜色在黄色和白色之间交替,每秒重绘一次图表
  - 重点国家弹窗
    - 检测到用户选择已经选择的国家且国家名字一致时,才会弹出对应的概览弹窗
- 注:图表闪烁占用了大量的资源,因为每次都要对图表进行重绘

##### 弹窗

- 弹窗设置

  - 由于canvas是原生组件,层级最高,故而需要用

    ```html
    <cover-view></cover-view>
    <cover-image></cover-image>
    ```

    来实现遮罩效果

  - 同时在弹窗中,采用**overflow:scorller**的方式来避免cover-view不允许嵌套scorllerview的弊端

- 语音播报

  - 引入微信插件 同声传译后,可以将文本转为一段云端的MP3语音
  - 同时用户点击后,将云端的MP3语音播放出来
  - 由于该插件的后台限制了一次播报的字符长度不超过1000,因此采用了分段播报的形式

#### 主包与分包

##### 背景

微信小程序为了提高加载速度,优化用户体验,强制规定每一个包必须小于2M才可以发布(含代码,图片,依赖等)

##### 解决方案

因此我们采用了官方推荐的分包机制,本小程序共有三个页面

- 世界地图(主页)
- 国旗选择页面
- 世界之最

其中,世界地图为主包,另外两个为分包,以保证每个包的大小小于2M,可以成功发布

##### 资源限制

由于资源限制在2M以内,故针对于图片资源,我们采用了两种方式存储:

- 本地存储,转为base64
  - 如页面背景图
- 服务器存储,提供url
  - 如世界之最,世界地图中的重点国家的各个图片等

##### 性能受限

###### 小程序本身原因

小程序本身就是运行在微信app中,因此性能本就和直接与操作系统打交道的app少了一些,这也是为什么微信官方要求每个包要小于2M,因为他们也知道性能是小程序的重要弊端之一

###### 本项目需求原因

本项目需要对地图持续闪烁,而每一次的闪烁都是需要对canvas画布进行重新渲染,每秒渲染一次,因此带来了大量的资源消耗,造成一定程度上的卡顿
